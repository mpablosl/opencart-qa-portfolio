# OpenCart QA Portfolio

## 💬 Metodologia de Estudo

Este projeto está sendo desenvolvido como um laboratório de estudos para minha transição de carreira para Qualidade de Software.

Durante o desenvolvimento, utilizo Inteligência Artificial como ferramenta de apoio para esclarecer conceitos, revisar os artefatos produzidos e simular interações com papéis comuns em um projeto de software, como Product Owner e QA Lead.

Toda a documentação de testes (análises, condições, objetivos, cenários, casos de teste e demais artefatos) é elaborada por mim e posteriormente revisada com o objetivo de identificar oportunidades de melhoria e aplicar boas práticas de Engenharia de Testes.

---

# 🎯 Objetivos de aprendizagem

Durante este projeto estou praticando:

- Análise da Base de Teste
- Interpretação de requisitos
- Identificação de Regras de Negócio
- Levantamento de dúvidas para Product Owner
- Identificação de Condições de Teste
- Definição de Objetivos de Teste
- Elaboração de Cenários de Teste
- Criação de Casos de Teste
- Execução de Testes Manuais
- Registro e documentação de Bugs
- Rastreabilidade entre requisitos e testes
- Organização de documentação técnica

Na próxima etapa do projeto também serão desenvolvidos:

- Automação de testes com Cypress
- Organização dos testes automatizados
- Evidências de execução
- Integração com GitHub

# 💡 Objetivo deste repositório

Este projeto não representa um sistema desenvolvido por mim.

Ele foi criado exclusivamente para demonstrar minha evolução prática na área de Qualidade de Software, documentando todo o processo de análise, planejamento, execução e, futuramente, automação de testes.

Meu foco é desenvolver não apenas habilidades técnicas, mas também a forma de pensar e atuar como um Analista de QA em projetos reais.
